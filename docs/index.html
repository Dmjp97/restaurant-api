Design

Layered Architecture

Every layer has a single responsibility. Controllers handle HTTP. Services handle business rules. Repositories handle data. No shortcuts.

🏗️
Repository Pattern
All DB queries behind interfaces. Services work against contracts — unit tests run with mocks, no real database needed.
⚙️
State Machine
Order status transitions are enforced server-side. pending→confirmed→preparing→ready→delivered. Invalid transitions return HTTP 422.
📡
Domain Events
OrderStatusChanged decouples mutations from side effects. Kitchen notifications fire via listeners, not inline code.

Security

Defence in Depth

Six independent layers. An attacker must bypass all of them.

🔑
JWT + Token Rotation
Access tokens expire in 15 min. Refresh tokens rotate on each use — stolen tokens revoked via Redis JTI blocklist.
👤
Role-Based Access Control
4 roles: superadmin · manager · cashier · kitchen. Every route declares required roles. Violations return 403.
🏢
Tenant Isolation
Users locked to their tenant at filter AND service layer. Cross-tenant reads are impossible even with a valid JWT.
🚦
Rate Limiting
60 req/min per IP backed by Redis. Atomic increments — accurate under concurrent PHP workers.
🛡️
Input Validation
CI4 validation on every mutating endpoint with explicit JSON body. SQL injection impossible via parameterised queries.
🔐
Password Hashing
Bcrypt via password_hash() auto-applied in UserModel beforeInsert hooks. Plain text never touches the DB.
