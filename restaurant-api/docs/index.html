Design Patterns

Clean Layered Architecture

Every layer has a single responsibility. Controllers never touch the database. Services never know about HTTP. Repositories never know about business rules.

🏗️
Repository Pattern
All DB queries behind interfaces — services work against contracts, enabling full unit testing without a real database.
⚙️
Service Layer
Business logic, state machine transitions, tenant checks and total calculations live here — never in controllers.
📡
Domain Events
OrderStatusChanged event decouples order mutations from notifications — listeners handle kitchen alerts and post-delivery flows.

/**
 * OrderService — all order business rules live here.
 * State machine enforces valid transitions only.
 */
class OrderService
{
    private const TRANSITIONS = [
        'pending'   => ['confirmed', 'cancelled'],
        'confirmed' => ['preparing', 'cancelled'],
        'preparing' => ['ready'],
        'ready'     => ['delivered'],
        'delivered' => [],
        'cancelled' => [],
    ];

    public function __construct(
        private readonly OrderRepositoryInterface   $orderRepo,
        private readonly ProductRepositoryInterface $productRepo,
    ) {}

    public function updateStatus(int $id, string $newStatus, ...): array
    {
        $order = $this->get($id, $tenantId);        // asserts ownership
        $this->assertValidTransition($order['status'], $newStatus);

        $this->orderRepo->updateStatus($id, $newStatus, $changedBy);

        // Dispatch domain event — listeners handle notifications
        Events::trigger(
            OrderStatusChanged::NAME,
            new OrderStatusChanged($id, $order['status'], $newStatus)
        );

        return $this->orderRepo->findById($id);
    }
}
/**
 * JWTAuthFilter — validates Bearer token on every protected route.
 * On success, injects the user into the request-scoped auth service.
 */
class JWTAuthFilter implements FilterInterface
{
    public function before(RequestInterface $request, $arguments = null)
    {
        $authHeader = $request->getHeaderLine('Authorization');

        if (empty($authHeader) || !str_starts_with($authHeader, 'Bearer ')) {
            return $this->unauthorized('Missing or malformed Authorization header.');
        }

        $token = substr($authHeader, 7);

        try {
            $payload = (new JWTHandler())->decode($token);

            if (($payload->type ?? '') !== 'access') {
                return $this->unauthorized('Invalid token type.');
            }

            $user = (new UserModel())->find($payload->sub);

            if (!$user || !$user['is_active']) {
                return $this->unauthorized('User not found or inactive.');
            }

            service('auth')->setUser($user); // inject into request scope

        } catch (\Exception $e) {
            return $this->unauthorized($e->getMessage());
        }
    }
}
/**
 * OrderRepository — all order queries centralised here.
 * Services never touch the query builder directly.
 */
public function create(array $data): int
{
    $this->db->transStart();  // atomic — items + timeline or nothing

    $this->db->table('orders')->insert([
        'tenant_id' => $data['tenant_id'],
        'status'    => 'pending',
        'total'     => $data['total'],
    ]);
    $orderId = $this->db->insertID();

    foreach ($data['items'] as $item) {
        $this->db->table('order_items')->insert([...$item, 'order_id' => $orderId]);
    }

    $this->recordTimeline($orderId, 'pending', $data['user_id']);
    $this->db->transComplete();

    if (!$this->db->transStatus()) {
        throw new \RuntimeException('Transaction rolled back.');
    }

    return $orderId;
}
Security

Defence in Depth

Six independent security layers. An attacker has to bypass all of them.

🔑
JWT with Token Rotation
Access tokens expire in 15 min. Refresh tokens rotate on every use — stolen tokens are invalidated immediately via Redis JTI revocation lists.
👤
Role-Based Access Control
4 roles: superadmin · manager · cashier · kitchen. Every route declares its required role. Violations return 403 — never 404 (no information leakage).
🏢
Multi-Tenant Isolation
Users are locked to their tenant_id at the filter layer AND the service layer. Cross-tenant reads are impossible — even with a valid JWT.
🚦
Rate Limiting
Sliding-window counter backed by Redis. 60 req/min per IP. Atomic increments prevent race conditions across concurrent PHP workers.
🛡️
Input Validation
CI4 validation rules on every mutating endpoint. Strict allow-lists for JSON fields. SQL injection impossible via parameterised query builder.
🔐
Password Hashing
Bcrypt via PHP's password_hash(). Auto-hashed in UserModel beforeInsert/beforeUpdate hooks. Plain passwords never hit the database.
